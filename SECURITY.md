# Security Policy

## Segnalazioni
Se trovi un problema di sicurezza nel kit (script, packaging, configurazioni):
- evita di aprire issue pubbliche con dettagli sfruttabili
- invia una segnalazione privata all’autore (canale concordato)

## Buone pratiche
- Non committare nel repo materiale sensibile (chiavi, keyring, report, output).
- Le Release devono includere checksum (SHA256) e, se possibile, firma del file checksum.
